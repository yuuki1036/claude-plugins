# Confidenceスコアリングガイド

## スコア定義

| スコア | 確信度 | 説明 |
|--------|--------|------|
| 0 | なし | 偽陽性、既存の問題 |
| 25 | 低 | 可能性あるが偽陽性かも。スタイル的な指摘で規約に明記なし |
| 50 | 中 | 実際の問題だがnitpickまたは実際にはまれ |
| 75 | 高 | 検証済みの問題。実際に発生する。重要度高い |
| 100 | 確実 | 確実にバグ。証拠あり。高頻度で発生 |

## フィルタリングルール

**報告閾値: ≥80**

### 除外対象（スコアに関わらず報告しない）

- 今回の変更で導入されたものではない既存の問題
- バグに見えるが正常なコード
- linterが検出するもの（ESLint, Prettier等）
- lint ignoreコメント付きのコード
- 些末なnitpick（スペース、改行等）

### スコア加算ルール

- CLAUDE.mdに明示的に記載あり: +20
- 複数エージェントが同一指摘を検出: +10
- git blameで過去に同様の修正あり: +15
- セキュリティ関連: +10

### スコア減算ルール

- テストコードでの指摘: -10
- コメントアウトされたコード: -20
- 自動生成コード: -30
- セッションコンテキストの設計判断と一致する指摘: -30
- セッションコンテキストの「スコープ外」に該当する指摘: -50

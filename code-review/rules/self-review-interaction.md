# セルフレビュー後の修正方針確認ルール

`self-review` スキルのレポート出力（Step 6）後に Step 7 として実行する。

## 共通原則

- **what/why を先に**: 選択肢より先に「何を聞いているか・なぜ確認が必要か」をテキストで明記する
- **Other（自由入力）を常に含める**: 選択肢が用意できる場面でも自由入力の逃げ道を用意する

## 修正方針の選択

指摘事項が1件以上ある場合のみ実行する（指摘0件なら「問題なし」で完了）。

レポート全文を出力し終えた直後に AskUserQuestion で確認する。

- **what**: 指摘事項への対応方針を選択する
- **why**: 今すぐ修正するか後回しにするかを明確にしておくと、コミット前の作業優先度が整理できる
- **選択肢**:
  1. すべて今すぐ修正する
  2. 重要な指摘だけ修正する（confidence >= 90）
  3. 修正はせず、このままコミットする
  4. その他（自由入力）

## 各選択肢の後処理

- **すべて修正**: 「修正を始めますか？」と一言確認してから着手する
- **重要のみ修正**: confidence >= 90 の指摘リストを再表示し、修正対象を確認してから着手する
- **このままコミット**: 完了とする
- **その他**: 自由入力の内容に従う
